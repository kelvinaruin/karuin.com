If a user machine is unable to resolve the new DNS server in your environment, it is more than likely due to the Cisco Umbrella RC agent caching the old DNS IP settings. You can confirm that this is the issue by running the command â€˜ipconfig /allâ€™ in the command prompt to check the DNS IP. If the IP is 127.0.0.1, then this is indeed the reason why this machine is not able to resolve new DNS paths, etc.

There is a workaround that will fix this problem. This fix also resolves the Azure VPN resolution problem, so if you are moving users to Azure VPN later, this will take care of that issue now and you will not have to do this when the new VPN client is deployed.
