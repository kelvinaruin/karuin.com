Scope

Provide steps on upgrading ESXi 6.5 or ESXi 6.7 to 7.x and/or ESXi 7.x to ESXi 8.x

Pre-Requisites

vCenter Server Appliance must be upgraded and or hosts migrated to new vCenter Server Appliance at same or higher revision than the version of ESXi on the host it is managing.

You can’t manage ESXi 7 hosts from earlier versions but you can manage ESXi hosts if VCSA is 7 or higher.

ESXi Upgrade Requirements

The Hardware of the Server running ESXi 6.7 must be compatible with ESXi 7.0. If an unsupported device is present, such as an unsupported CPU or disk controller, the ESXi upgrade will not be applied to the Server. See VMware Hardware Compatibility Guide for verification.

No VMs on an ESXi host must be running when you start the ESXi upgrade process.

There must be enough free storage space for the ESXi upgrade.

If an ESXi host is managed by vCenter Server, the vCenter version must be the same as the target version of ESXi after the upgrade or higher. VMware vCenter 7.0 can manage ESXi 7.0, ESXi 6.7, and ESXi 6.5, but vCenter 6.7 cannot manage ESXi 7.0 hosts.

NOTE: You must upgrade vCenter server first or build a new vCenter server and migrate your hosts.

If any VMware add-ons or plugins are installed, check their compatibility using the VMware Compatibility Matrix.

Determine and verify if standard VMWare image can be used or if a custom OEM image is installed.

For earlier versions 5.x or 6.0; you must upgrade to a supported upgraded version (i.e., 6.5 or 6.7 first) before proceeding to 7.x

Ensure how DNS is configured in your environment to verify that powering off any DNS servers on the server you’re upgrading does not take internet access down for the site you’re upgrading.

NOTE: If you have more than one DNS server, ensure that you are only powering off one at a time, or be sure to perform this work onsite to avoid losing access to your host.

ESXi Upgrade Methods

This SOP covers two main methods for a VMware ESXi upgrade. Although there is a third, when you could boot off the ISO via iLO/iDRAC, and instead of installing ESXi, you choose to upgrade. Third method is not covered in this document.

Upgrade by using ESXCLI, the command line interface. This method can be used for VMware ESXi 6.7 to 7.0 upgrade for standalone ESXi hosts and ESXi hosts managed by vCenter.

NOTE: You must use this if your vCenter server lives on the host you’re upgrading.

Upgrade by using Lifecycle Manager. This method is applicable only for ESXi hosts managed by vCenter VCSA. VMware Lifecycle Manager for VMware vSphere 7.0 is a modified VMware Update Manager that was used in vSphere 6.7. This is the preferred method for upgrading ESXi hosts managed by vCenter and allows you to upgrade multiple ESXi hosts at once by using the same image.

If the ESXi nodes are managed by vSphere VCSA and are within the same cluster with the ability to vmotion, you can limit downtime and impact to the Client as you upgrade each node. Otherwise, you must schedule downtime for the VMs running on the node.

NOTE: vMotion is not included with Essentials licensing and is an additional license.

The workflow of upgrading ESXi 6.7 to ESXi 7.0, ESXi 7.0 Update 1, ESXi 7.0 Update 3, and other versions/builds of ESXi 7 is similar. This approach is also proper for upgrading ESXi 7.0 to ESXi 7.0 Update 3, ESXi 7.0 Update 3l, etc. Remember that the names of installation files and update profiles differ in this case.

Choose one of the listed upgrade paths.

For Standalone ESXi, use the CLI upgrade method.

There is an offline method and an online method.

For ESXi managed by VCSA, it might be easier to create a baseline and attach it to the designated node(s) or cluster(s).

NOTE: You can’t do this on a host that is running VCSA. VCSA would need to be migrated to another host in the same cluster where the existing subnet exists for the IP to be reachable.

Choose based on your deployment needs.

You may need to use both if VCSA lives on a single host and the subnet range is different on the other hosts/datacenters.

ESXi Upgrade by using the CLI.

Upgrading ESXi 6.7 to ESXi 7.0 Update 3 using ESXCLI is straightforward, but you should still have an ESXi backup before upgrading. You can roll back to the last version using the backup if anything goes wrong. Please verify you have successful backups for the VMs managed by the host.

Enable ESXi Shell and SSH to access the command line interface on the ESXi host. You can do this by using one of two methods:

In the ESXi direct console (DCUI), open System customization, and in Troubleshooting options, hit Enable ESXi Shell and Enable SSH. This configuration enables SSH and ESXi Shell permanently until you manually disable them.

Enter the IP address of your ESXi host in a web browser. Log in to the VMware Host Client by using your ESXi root credentials. In the web interface of the VMware Host Client, go to Manage > Services.

Start these services.

TSM

TSM-SSH

These services will be enabled until ESXi reboot if you started the services in VMware Host Client

After enabling SSH you may login using a SSH Client, i.e., Putty, or latest version of PowerShell has SSH tools built in.

Check the current version of ESXi in the ESXi Direct Console by logging into the VMware Host directly and going to Help > About.

ESXi Shell by using the vmware -v command:

If the Client has multiple host(s), and a vMotion license, use VMotion to migrate all VMs off of the host being updated.

If the Client does not have a vMotion license and multiple hosts, gracefully shut down each VM.

Put the host in maintenance mode by right-clicking on the host and selecting maintenance mode.

ESXi upgrade by using ESXCLI is divided into two types:

Offline method: manually download either the full ISO or the offline Bundle from the VMware website, upload the file to the ESXi datastore, and use this Bundle for the ESXi upgrade.

Online method: you should use VMware online repositories to download and install new packages for an ESXi profile update.

Log onto a Windows Server with access to the Virtual Environment at the client site.

Browse to the ESXi UI in the web browser.

Upload the downloaded ZIP or ISO file to the ESXi datastore.

Go to Storage > Datastores, select the datastore (datastore1 in this example), and click Datastore Browser.

Create or select an existing directory to store bundle files in the datastore browser. We created the ESXi7-bundle directory to store the offline Bundle previously downloaded. Click Upload to browse locally on server for the ISO/ZIP bundle file to complete the upload.

Verify there are no VMs in the running state on this ESXi host before starting the ESXi upgrade process.

If the Client has multiple host(s) and a vMotion license, use VMotion to migrate all VMs off the host being updated.

If the Client does not have a vMotion license and multiple hosts, gracefully shut down each VM.

Place the host in maintenance mode by right-clicking on the host and selecting maintenance mode.

Alternatively, you can use ESXCLI to place the node into maintenance mode.

esxcli system maintenanceMode set –enable true

Obtain the image profile name of the offline zip file you downloaded.

esxcli software sources profile list -d /vmfs/volumes/datastore1/ESXi7-bundle/VMware-ESXi-7.0U3k-21313628-depot.zip

Run the update command to upgrade ESXi 6.7 to 7.0 Update 3:esxcli software profile update -p ESXi-7.0U3k-21313628-standard -d /vmfs/volumes/datastore1/ESXi7-bundle/VMware-ESXi-7.0U3k-21313628-depot.zip

“Datastore1” is the name of the available datastore: Depending on the implementation, this name could be different.

Offline package names could be different depending on the upgrade or custom image.

VMware-ESXi-7.0.3-20842708-HPE-703.0.0.11.2.0.9-Jan2023-depot.zip

Wait until you see the command output.

After the successful execution of the command, you can see a list of updated packages (VIBs – VMware installation bundles) in the console output.

Reboot the ESXi host to finish the ESXi upgrade:

Post Upgrade Tasks

You need to add the new license to VCSA and apply the new license to the upgraded host upon completion. By default, you will have a evaluation license only applied.

Perform the latest critical updates.

Critical updates may have additional driver updates that have been released separately for the Update X package.

The same methods outlined here for upgrading the hosts by CLI or VCSA can be used for these updates to be applied. Again, if VCSA is on a single host then CLI method is the only option.

NOTE: Product updates can be downloaded for depot ZIP files from the https://my.vmware.com customer care account.

All VMs will need their VMTools upgraded.

This can be done using the automatic method without any options via VCSA.

NOTE: This will cause your VMs to reboot so you will want to complete this during your maintenance window.

Online method for ESXi upgrade via ESXCLI.

Again, make sure you have successful backups.

Verify there are no VMs in the running state on this ESXi host before starting the ESXi upgrade process.

If the Client has multiple hosts, use VMotion to migrate all VM’s off the host being updated.

If the Client does not have multiple hosts, then gracefully shutdown each VM.

Place the host in maintenance mode by right-clicking on the host and selecting maintenance mode.

Alternatively, you can use ESXCLI to place the node into maintenance mode.

esxcli system maintenanceMode set –enable true

When using the online method, you should configure internet access for the ESXi host you want to upgrade and manually place the host in maintenance mode.

List the current image profile and contained packages:

esxcli software profile get.

Enter ESXi into maintenance mode:

esxcli system maintenanceMode set –enable true

Allow HTTP traffic by editing the settings in the built-in firewall on the ESXi host:

esxcli network firewall ruleset set -e true -r httpClient

List profiles provided by VMware for ESXi 7.0 Update 3 in VMware repositories:

esxcli software sources profile list -d https://hostupdate.vmware.com/software/VUM/PRODUCTION/main/vmw-depot-index.xml | grep ESXi-7.0U3

A software depot is a collection of VIBs and image profiles. It has a hierarchy of files and directories accessed via HTTPS.

Let’s use this profile as an exampleESXi-7.0U3k-21313628-standard

Here is an online site to find the latest image profiles and patches:

You can check which packages will be installed before running the ESXi upgrade command: with –dry-run at the end.

esxcli software profile update -d https://hostupdate.vmware.com/software/VUM/PRODUCTION/main/vmw-depot-index.xml -p ESXi-7.0U3k-21313628-standard --dry-run

Run the update process:

esxcli software profile update -d https://hostupdate.vmware.com/software/VUM/PRODUCTION/main/vmw-depot-index.xml -p ESXi-7.0U3k-21313628-standard

Wait for a few minutes until the upgrade process is finished.

Choose esxcli software profile update instead of esxcli software profile install to prevent overwriting existing packages, drivers, and configuration.

Note: If you see the error message:

Got no data from process: LANG=en_US.UTF-8 /usr/lib/vmware/esxcli-software profile.update  -d https://hostupdate.vmware.com/software/VUM/PRODUCTION/main/vmw-depot-index.xml" -p "ESXi-7.0.0-15843807-standard"

Check whether there is a persistent scratch partition. A persistent scratch partition may not exist if you run ESXi from a USB flash drive.

If everything is correct, reboot the ESXi host for the changes to take effect.

reboot

Exit maintenance mode with the command:

esxcli system maintenanceMode set --enable false

ESXi Host Upgrade in vCenter VCSA

Open VMware vSphere Client in your web browser to access vCenter.

There are two ESXi 6.7 hosts in this example, managed by vCenter 7.0.3. Shut down virtual machines on the ESXi hosts that you want to upgrade or migrate those VMs to other ESXi hosts.

Let’s upgrade the ESXi host with the IP address 192.168.101.65.

Select the needed ESXi host and check the current hypervisor version. As you can see in the screenshot below, the current version of our ESXi host before the upgrade is 6.7.0, build 10302608. You can select a cluster and a data center to upgrade multiple ESXi hosts.

Open Lifecycle Manager. You can do it by using different methods:

Click Home, and in the left pane (navigation pane), click Lifecycle Manager OR

Click Menu at the top of the vSphere Client window and hit Lifecycle Manager.

Once you have opened VMware Lifecycle Manager in vSphere Client, select the Imported ISOs tab, and click Import ISO.

Hit Browse and select the ESXi 7 Update 3 installation ISO image file. You can download the ESXi installation image from the VMware website.

Find the Clients login information in IT Glue

If using Custom Image, select the appropriate ISO.

Example VMware-ESXi-7.0.3-20842708-HPE-703.0.0.11.2.0.9-Jan2023.iso

Once you have selected the ISO file, the importing process starts. In this example, the VMware-VMvisor-Installer-7.0U2-17630552.x86_64.iso file is used.

After importing the ISO file, you can see the ESXi 7.0 Update 2 installation image in the Imported ISOs tab.

Click New baseline to create a new baseline.

Name and Description. Enter the baseline name and description, for example, ESXi 7.0 Update 2 and ESXi 7.0 Update 2 baseline.

Select ISO. Select the ESXi 7 ISO image you have uploaded before (VMware-VMvisor-Installer-7.0U2-17630552.x86_64.iso). The displayed name is based on the corresponding ESXi profile (ESXi-7.0.2-17630552-standard).

Summary. Review your selections before finishing creating the baseline.

A baseline has now been created.

Go back to the Hosts and clusters screen, select the ESXi host that must be updated (192.168.101.65 in our case), and click the Updates tab in the top right corner of the interface.

When the baseline has been created, you must attach that baseline to the ESXi hosts you want to update. Remember, we will update a single ESXi host in this walkthrough.

Hit Baselines, scroll down, and find the Attached Baselines section, then click Attach and in the open menu, click Attach Baseline or Baseline Group.

Select the ESXi7.0 Update 2 baseline you have created before and click Attach. If any other baselines are attached, remove them by selecting them and clicking Detach.

Now you must check compliance. This step is needed to check whether the ESXi host has installed the update. If you see the non-compliant status, the update is not installed on your ESXi host. Click Check Compliance.

As you can see in the following screenshot, the ESXi status after checking is non-compliant, which means that you can install the needed update and upgrade ESXi 6.7 to ESXi 7.0 Update 2.

Pre-check remediation checks for any compliance issues that may occur after installing updates and the ESXi upgrade.

As you can see in the screenshot below, there are no issues after running the remediation pre-check. Hit Done and go to upgrade the ESXi host.

In the Attached Baselines section, select your ESXi 7.0 Update 2 baseline created before and click Remediate to start the ESXi 6.7 to 7.0 upgrade process.

A pop-up screen appears. Read the End User License Agreement (EULA) and select the I accept the terms and license agreement checkbox. Hit OK to continue.

Ensure your ESXi host is selected and hit Remediate to upgrade ESXi 6.7 to 7.0. Don’t forget that virtual machines must be powered off or migrated from the ESXi host that is being upgraded.

You can see the ESXi upgrade progress in the Recent task toolbar, which can be expanded at the bottom of the VMware vSphere Client interface. The ESXi host is automatically entered into maintenance mode to install the upgrade.

When the ESXi upgrade process is finished, maintenance mode is turned off for the host, and the upgraded ESXi host is ready to work. You can select the ESXi host in the Hosts and Clusters view and check the version after the ESXi upgrade. As you can see in the screenshot below, the version is now ESXi 7.0.2, 17630552.

If a license expires after an ESXi 6.7 to 7.0 upgrade on the upgraded host, you should apply a vSphere 7.0 license.

If you need to upgrade an ESXi host on which a vCenter VM (a virtual appliance that is also known as VCSA) is running, you can use one of the following approaches:

Migrate the vCenter VM to another ESXi host and upgrade the source ESXi host.

Shut down the vCenter VM and other VMs running on the ESXi host and upgrade ESXi using ESXCLI, as explained in the first part of this blog post. Note that SSH access must be enabled, and the lockdown mode must be disabled (you must be able to manage an ESXi host directly without vCenter).

Note: If none of the above ESXi upgrade methods is suitable for your case, you have another option:

You can write the ESXi ISO installation image to a DVD or USB flash drive, boot from that medium, run the ESXi 7.0 or ESXi 7.0 Update 2 installer, and select the disk where ESXi 6.7 is installed as the destination disk. The upgrade is performed in the interactive mode in this case.

Conclusion

VMware ESXi upgrade in ESXCLI, a command-line interface, is a universal method that can be used to upgrade standalone ESXi hosts and ESXi hosts managed by vCenter if it is not restricted by vCenter configuration. If you use vCenter to manage many ESXi hosts, consider ESXi upgrade using vCenter Lifecycle Manager, a component of vCenter 7.0 and a successor of VMware Update Manager. Having the ESXi and VM backups would be best before starting an ESXi upgrade.
