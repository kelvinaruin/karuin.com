Intune Migration with ProfWiz

This article documents how to migrate an existing on-premises Active Directory user profile to an Entra ID user profile using ForensiT Profile Wizard. The goal is to preserve the user’s existing local profile data while transitioning the workstation to Entra ID.

Category: Intune & Endpoint Platform: Windows Identity: Entra ID Tool: ForensiT ProfWiz Estimated Read: 10–15 min

Use Case

Use this process when a workstation has been removed from an on-premises Active Directory domain and joined to Entra ID, but the user’s existing local profile data must remain available under the new Entra ID sign-in.

This process is intended to retain user profile data such as:

  • Files
  • Desktop, taskbar, Start Menu, and system tray shortcuts/settings
  • Certain application settings stored in the user profile
The best time to perform this migration is immediately after removing the PC from the on-premises domain and joining it to Entra ID, before the user signs in with the Entra ID account. This reduces the risk of overwriting or losing data in a newly created profile.

Prerequisites

Before starting, confirm the workstation is in the following state:

  • The PC has been disjoined from the on-premises domain.
  • The PC has been joined to Entra ID.
  • The target user has not yet signed in with their Entra ID UPN on this workstation.
  • You are signed in using a local administrator account such as IMPACTLOCAL.
  • You are not signed in using a domain or Entra ID account while running the migration.

Example migration mapping:

Source profile: <CLIENTDOMAIN>\idomaintest
Target account: <username>@<clientdomain.com>

Required Files

Download or stage the ProfWiz project archive before beginning the migration. In the source workflow, the archive is named ProfWIZ.zip and is stored in the GitHub documentation repository.

  1. Sign in as the local administrator account.
  2. Download the required ProfWIZ.zip archive.
  3. Create a new folder called Impact on the C drive.
  4. Extract the archive to C:\Impact\ProfWIZ\.
C:\Impact\ProfWIZ\

Migration Process

Step 1 — Open PowerShell as Administrator

Open an elevated PowerShell session while signed in with the local administrator account.

Step 2 — Change to the ProfWiz Working Directory

Change to the ProfWiz working directory.

cd C:\Impact\ProfWIZ

Step 3 — Set the PowerShell Execution Policy

Run the following command and press Enter:

Set-ExecutionPolicy RemoteSigned

If prompted, press Y and then Enter.

Step 4 — Run the Azure AD User Preparation Script

Run the preparation script from the working directory:

.\Save-AzureADUser.ps1

Press Y and Enter for prompts that appear during script execution.

Step 5 — Sign In with the Target Microsoft 365 Account

When prompted, sign in using the Microsoft 365 account that the profile is being migrated to.

Step 6 — Approve Microsoft Graph Consent if Prompted

If prompted for Microsoft Graph consent, check the consent box and select Accept.

Step 7 — Verify the ForensiT Azure ID XML File

Confirm the following file was created after the script completes successfully:

C:\Impact\ProfWIZ\ForensiTAzureID.xml
If the file is not visible, enable file name extensions in File Explorer and confirm the script completed without errors.

Step 8 — Run the Profile Wizard Installer

Run ProfWiz.msi. If prompted to run as administrator, select Yes.

Step 9 — Start the Profile Wizard

In the Profile Wizard interface, click Next.

Step 10 — Select the Source Profile

Select the source account profile that should be migrated. This is usually the user’s old domain profile.

The source account may display as a SID or generic identifier instead of the original domain username because the workstation has already been disjoined from the domain. Use the profile path to identify the correct user profile folder.

Step 11 — Enter the Target Entra ID UPN

Confirm the domain and Azure AD/Entra ID options are selected as required by the tool. Enter the target user’s email address/UPN and click Next.

<username>@<clientdomain.com>

Step 12 — Run the Migration

Profile Wizard will begin linking the existing profile to the target Entra ID account. This should usually complete quickly because profile data is not being physically copied to a new profile.

When the process completes, click Next, then click Finish.

The workstation will restart after clicking Finish. Confirm there are no open files, active installs, or unsaved work before completing this step.

Step 13 — Sign In with the Entra ID User Account

After the workstation restarts, sign in using the user’s Entra ID UPN. The existing profile data should now be retained under the Entra ID account context.

Validation

  1. Log out of the current account.
  2. Sign in with the new user account using the user’s Entra ID UPN.
  3. Validate that profile data was retained.
  4. Confirm expected files, desktop items, shortcuts, and application settings are present.
  5. If data is missing, review the old profile path and manually copy required items as needed.

After validation is complete, proceed to the next workstation scheduled for migration.

Known Notes

  • The old domain profile may appear as a SID or orphaned identifier after the domain disconnect.
  • The profile path is the most reliable indicator when selecting the source profile.
  • This migration should be performed before the user signs in and starts working in the new Entra ID profile.
  • The workstation restarts after the Profile Wizard process is completed.
