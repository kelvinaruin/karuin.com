Configuring Mimecast with O365

Status: Imported Source: DOCX Images: 19 Last Updated: 2026

Mimecast Domain Setup

Access the Mimecast Administration Console via https://login-us.mimecast.com/administration/

Add all domains that will be utilized for email, including TENANT.onmicrosoft.com to Mimecast

To add domains, navigate to Administration > Directories > Internal Directories

Click on the Register New Domain button

Enter the domain name at the bottom of the page to complete step 1

Press the Get Verification Code button

Add the required TXT record to the domain via the public DNS registrar such as Cloudflare or GoDaddy

Once the TXT record exists in the public DNS, click the Validate button at the bottom of the page

Point Mimecast to 365 MX record

If all information was entered correctly, the domain will be successfully validated as displayed on the Summary page

Keep this page open as it will be needed after the Azure configuration in the next section

Office 365 Connection Filter Configuration

Log on to the Microsoft 365 Admin Center

Click on the Security menu item

Click on the Policies & rules menu item

Select Threat policies

Select Anti-spam

Select the Connection filter policy, then select Edit connection filter policy

Give the filter a Description

Click on Edit connection filter policy

On the Always allow messages from the following IP addresses or address range, add the Inbound Mimecast IP Ranges

The updated Mimecast IP information can be found at the following link: https://community.mimecast.com/s/article/Adding-Network-Ranges-to-Office-365

Exchange 365 Outbound Connector Configuration

Login to the 365 Exchange Admin Center

Go to Mail Flow > Connectors and click on the Add a connector

Select Connection from Office 365 and connection to Partner Organization

Name the connector O365 Outbound to Mimecast, then choose Next

Select Only when I have a transport rule set up that redirects messages to this connector, then press Next

Select Route email through these smart hosts, then enter the two entries for Mimecast

us-smtp-o365-outbound-1.mimecast.com

us-smtp-o365-outbound-2.mimecast.com

Make sure Always use Transport Layer Security (TLS) to secure the connection (recommended) and Issued by a trusted certificate authority (CA) are selected, then press Next

Enter a validation email account

Press Validate

Press Next

Press Save

Office 365 Journaling with Mimecast Configuration

Create an Office 365 Journaling Contact

Logon to Exchange 365 Administrator portal

Go to Recipients > Contacts

Click Add a mail contact

Fill out the basic information

First Name: Journaling

Last Name: Mimecast

Display Name: Journaling for Mimecast

Alias: Journaling

External Email Address: journaling@journal.domain.com

Press Next again

Review the information entered and press Create

Create an Office 365 Journaling Send Connector

Logon to Exchange 365 Administrator portal

Go to Mail Flow > Connectors

Click Add a connector

Select From Office 365

Select To Partner Organization

Press Next

Name connector Office 365 to Mimecast Journaling

If you are ready to enable the connector, leave the Turn it On box checked, if not uncheck it

Click Next

Select Only When Email Messages are Sent to These Domains, type the domain *.journaling.DOMAIN.com and click the + button

Press Next

Select Route email through these smart hosts and enter the below Mimecast hostnames

us-smtp-journal-1.mimecast.com

us-smtp-journal-2.mimecast.com

Press Next

Select the following security restrictions

Always use Transport Layer Security (TLS) to secure the connection

Issued by a trusted certificate authority (CA)

Press Next

Enter the journaling email used to create the journaling contact

journaling@journal.domain.com

Press Validate

Press Save

Create an Office 365 Journaling Rule

Login to the Compliance 365 Administrator Portal (compliance.microsoft.com)

Go to Data lifecycle management > Exchange (legacy) > Journal rules

Click New rule

Enter your journal contact email into the Send journal reports to field

journaling@journal.domain.com

Name rule O365 Journal Rule

Select Journal messages sent or received from Everyone

Select All messages for Type of message to journal

Press Next, then review the information entered

Press Submit if correct

Configuring a Journal Definition in Mimecast

Login to the Mimecast Administration Console

Navigate to Administration > Services > Journaling

Click New Journal Service Definition

Enter a name in the description

Select SMTP for Transport type

Complete the Connection Properties Service Email Address with the contact email created in O365

journaling@journal.domain.com

IP Ranges can be left blank. If you need to add O365 IP ranges here, you will have to contact Mimecast support

Select Use SMTP Authentication

Enter password

Leave the default value in Initial Process Delay and Delivery Wait Attempts

Period of Inactivity Allowed: Default is 180 mins

Journal Type: Exchange

Encrypted: leave enabled

Rest can be left alone

Save and exit

Configuring Journaling Subdomain in Mimecast

Login to the Mimecast Administration Console

Navigate to Administration > Directories > Internal Directories

Select Add Subdomain button

Domain: journal.domain.com

Accept all inbounds for this domain

Leave anti-spoofing unselected

Conclusion

Before mail will route through Mimecast, the following needs to be configured:

Public DNS MX record should point to Mimecast

Records can be obtained from: https://community.mimecast.com/s/article/Office-365-Configuring-Outbound-Delivery-Routing-1430192975

Use the following:

us-smtp-inbound-1.mimecast.com

us-smtp-inbound-2.mimecast.com

All O365 connectors created will need to be turned on

Azure AD Sync will need to be enabled in Mimecast

Public DNS domain SPF record will need to be updated to include Mimecast
