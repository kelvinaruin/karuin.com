FRS to DFSR

Status: Imported Source: DOCX Images: 6 Last Updated: 2026

Overview

Older Server versions used FRS to replicate changes to the SYSVOL directory to all domain controllers. Server 2008 R2 deprecated FRS and instituted DFSR as the preferred method for keeping folders synchronized across multiple domain controllers.

SYSVOL is a folder shared by the domain controller to hold its logon scripts, group policies and different items associated to AD. All the domain controllers in the network will replicate the content of the SYSVOL folder. The default path for SYSVOL folder is %SystemRoot%\SYSVOL.

Windows Server 2003 and 2003 R2 make use of File Replication Service (FRS) to duplicate SYSVOL folder content to different domain controllers. However, Windows Server 2008 and later makes use of Distributed File System (DFSR) for the replication. DFRS is more efficient than FRS. Adding Windows 2008/R2 into an existing 2003 Domain DID not automatically migrate from FRS to DFSR, nor will migrating FSMO roles off legacy Servers migrate SYSVOL replication from FRS to DFS. This document will outline the steps necessary to migrate from FRS to DFSR using the Dfsrmig.exe utility.

** Note should you try to add a Windows 2016 RS3 or Windows 2019 Server as a domain controller that is currently using FRS ** (it will not fully work until you migrate from FRS to DFSR)

Ensure All domain controllers have enough free disk space:

The DFSR migration process copies the contents of SYSVOL to a parallel folder called SYSVOL_DFSR, and then shares out that copy during the Redirection phase. This means that on the volume where your SYSVOL exists on domain controllers - typically the C: drive - you need at least as much free space as the size of the current SYSVOL folder, plus a 10% fudge factor. For instance, if your current SYSVOL folder is 2GB (an unusually large SYSVOL), you should ensure that at least 2.2 GB disk space is free on the same volume. Most SYSVOL are only a few hundred MB or less.

FRS to DFSR

Overview

Before moving into the configurations, we must investigate the migration levels.

There are 4 stable states going together with the 4 migration phases.

1) State 0 â€“ Start

2) State 1 â€“ Prepared

3) State 2 â€“ Redirected

4) State 3 â€“ Eliminated

State 0 â€“ Start

Initiating this state, FRS will replicate SYSVOL folder among the many domain controllers. It is especially important to have an updated copy of SYSVOL before the migration process begins to keep away from any conflicts.

State 1 â€“ Prepared

On this state, while FRS continues replicating SYSVOL folder, DFSR will replicate a replica of SYSVOL folder. It will be positioned in %SystemRoot%\SYSVOL_DFRS by default. However, this SYSVOL will not respond to some other domain controller service requests.

State 2 â€“ Redirected

In this state, the DFSR copy of SYSVOL begins to respond to SYSVOL service requests. FRS will proceed with the replication of its own SYSVOL copy, however, it will not be involved with the SYSVOL replication.

State 3 â€“ Eliminated

In this state, DFS Replication will continue its replication and servicing SYSVOL requests. Windows will delete original SYSVOL folder users by FRS replication and cease the FRS replication.

To migrate from FRS to DFSR you must go from State 1 to State 3. This step cannot be reversed. And each stage may take some time to complete.

Migration Steps:

Prepared State

Log in to domain controller as Domain admin or Enterprise Admin

Launch PowerShell console.

Type dfsrmig /setglobalstate 1 and press enter.

Type dfsrmig /getmigrationstate to verify all domain controllers have reached a prepared state.

Redirected State

Log in to domain controller as Domain admin or Enterprise Admin

Launch PowerShell console.

Type dfsrmig /setglobalstate 2 and press enter.

Type dfsrmig /getmigrationstate to verify all domain controllers have reached redirected state.

Eliminated State

Log in to domain controller as Domain admin or Enterprise Admin

Launch PowerShell console.

Type dfsrmig /setglobalstate 3 and press enter.

Type dfsrmig /getmigrationstate to verify all domain controllers have reached eliminated state.

Completion

This completes the migration process and to verify the SYSVOL share.

From command prompt type net share

verify sysvol added sysvol_DFRS.

reverify repadmin /ReplSum

Verify on all the domain controllers the old FRS service is stopped and disabled.
