Fix Azure VPN Client Issues due to Umbrella

If a user is unable to connect to Azure VPN via Azure VPN client, and the issue is hostname not resolved, it is more than likely that Umbrella is the culprit. There is a workaround that will fix this problem.

Check to see if the Umbrella Roaming client is installed.

You can see this by checking services and seeing if a service called Umbrella_RC exists.

If installed, uninstall Umbrella Roaming Client via the control panel or settings.

NOTE: There is an AMP for uninstalling Umbrella, if you have a lot of machines and not working on just one. The AMP for uninstalling is Impact - Uninstall Umbrella AMP. This can be run via a scheduled task in INC to address more than one machine.

Confirm the client is uninstalled by checking services for Umbrella_RC service. It should now be gone.

Set the DNS to the correct IP if needed. Do not leave the 127.0.0.1 entry and update to the domain controller or public DNS.

Validate you are now able to access DNS entries, such as the file server mapped drives with the Umbrella client removed and the correct DNS settings set on the client machine.

If everything works as expected, continue to the next step. If not, you may need to perform additional troubleshooting.
