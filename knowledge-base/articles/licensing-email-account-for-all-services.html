Overview

ISSUE: Licensing for various vendors and MS VLSC for customers is tied to old IT employees or individual accounts.

STANDARDIZATION: Several clients are not consistent within their organizations for licensing emails, access to MS open licensing agreements, server licensing, and software licensing time-sensitive email notifications for renewals not landing into the hands that need them.

Impact FNEs use the Impact client distribution groups for some of these licensing areas. This makes it challenging to manage and handoff in the event of a client offboarding when Impact client distribution groups are no longer active email addresses.

This also creates reactive rather than proactive support when licenses expire or renewal periods are missed.

RESOLUTION: Use a single account across the board hosted within the clientâ€™s email server.

Create a standard user account for all clients to redirect licensing renewals and vendor application contracts to use a client-shared mailbox within their email server.

Enable mail forwarding on this account to go to a new internal IT Management distribution group on the clientâ€™s email server, with members being the Impact client distribution group and executive officers for the client company.

BACKGROUND: Microsoft VLSC now leverages integration with existing O365 accounts. The account needs to be an actual user account. To address this, instead of creating a shared mailbox directly on the email server, we create a user account, assign a license, convert it to a shared mailbox, and remove the license.

Assumptions

You have already migrated email to O365 or have an existing O365 email server.

You have access to an existing MS VLSC account for customer open licensing.

You have access to all vendor licensing for the existing line of business applications.

The Impact client distribution group can receive external emails.

You have O365 Global admin rights.

You have an available license for the initial account/mailbox creation.

You can steal the impactadmin account license temporarily if no spares exist.

Licensing Account Creation

Create a new user service account in AD if AAD Connect is configured or a cloud-only account with the password set never to expire. Be sure to use a randomly generated complex password.

Licensing@clientdomain.com

Document credentials for the account in IT Glue.

Assign an O365 or M365 license with Exchange Online Plan for regular user mailbox creation on the email server.

Convert the mailbox to a shared mailbox in Exchange Admin Center.

Remove license from user account.

Name: Client Name Impact DL

Hide the contact from the Global Address List

Email: Use the MIT Client distribution group name and email address.

Create New IT Management Distribution Group

Create a new distribution group as a cloud-only account.

Name = IT Management

Email Address = itmanagement@clientdomain.com

Assign members and include stakeholders.

Use the new Impact Client Distribution Group contact and the key stakeholders. This could also include the CEO, President, CIOs, and other executives who wish to be included in licensing emails.

Ensure the distribution group is set to allow external senders.
