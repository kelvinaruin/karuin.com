Modify XML file for Office Removal

Newer versions of Office (Office 2019 and above) cannot be removed through the same method that is created from the ODT configuration as they are not installed using the MSI method. The area below needs to be added to the XML to remove the newer versions of Office. See full XML example at the end of the document. This link does have the list of all of the Product IDs

https://learn.microsoft.com/en-us/microsoft-365/troubleshoot/installation/product-ids-supported-office-deployment-click-to-run

What needs to be added:

<Remove All="FALSE">

<Product ID="ProPlus2019Volume" >

</Product>

</Remove>

Download Office Software

Open an elevated Command Prompt or PowerShell console window.

Navigate to the ODT share you created earlier or update the command in 3a to point directly to your share path.

Run the following command.

Command Prompt:

ODT.EXE /download ClientConifgCDN.xml or

\\UNCsharepath\odt.exe /download ClientConfigCDN.xml

PowerShell:

.\ODT.exe /download .\ClientConfigCDN.xml or

\\UNCsharepath\.\odt.exe /download ClientConfigCDN.xml

NOTE: If running ODT.exe from a NAS or UNC path with option 2 of the command, the Office folder will be created, and files downloaded to the default path displayed in your command prompt. Ex. C:\Users\username. You will need to then copy the downloaded files to the NAS or UNC path.

It will take a bit of time to download the software, this depends on the WAN bandwidth.

Once finished, a folder called Office will be created in the ODT share.

You are ready to deploy the software.

Deploy Software Manually

You can manually deploy the software on a user workstation by browsing to the \\SERVERFQDN.COM\ODT share via CMD or PowerShell console.

You will then want to run the following command to uninstall any old versions of office and install the Office 365 current version with the settings specified in the XML created earlier.

Run the following command.

Command Prompt:

ODT.EXE /configure ClientConifg.xml

\\UNCsharepath\odt.exe /configure ClientConfig.xml

PowerShell:

.\ODT.exe /configure .\ClientConfig.xml

\\UNCsharepath\.\odt.exe /configure ClientConfig.xml

Office will silently install within 10 minutes or so.

Deploy Software to all Workstations via INC RMM

Create a domain service account that is a member of the Domain Administrators group.

Open the appropriate INC instance and navigate to the appropriate client.

Under Actions, select Push Third Party Software.

Rename the Task Name at the top of the page to something that makes sense.

Select Custom Credentials and enter the service account username and password you created in step 1.

Select From My Computer under Location.

Type ODT Office Deployment under Name.

Fill in a description.

Click Browse next to File Name and locate the ODT.EXE in the ODT share.

Enter the following for Command Line Parameters:

ODT.exe /configure \\SERVERFQDN.com\ODT\ClientConfig.XML

Click on Targets and select all the workstations you would to deploy office to.

Click Save.

Your deployment is on its way to the workstations selected.
