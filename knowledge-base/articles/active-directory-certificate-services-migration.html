Active Directory Certificate Services Migration

Status: Imported Source: DOCX Images: 18 Last Updated: 2026

Overview:

This document serves as a guide for migrating the Active Directory Certificate Services (AD CS) from a Source Server (Current CA host) running Windows Server OS 2008 R2 - 2022 to a Destination Server (New CA host) running Windows Server 2016 â€“ 2022. The process involves several critical steps to ensure a seamless migration while preserving the integrity of the certificate authority (CA) database, configuration, and certificate templates. These steps include backing up the CA database and registry settings, documenting and manually reconfiguring certificate templates, uninstalling the CA service on the old server, installing and configuring AD CS on the new server, restoring the CA database and registry, and reissuing the certificate templates. Proper execution of each step ensures minimal disruption and a smooth transition to the new server.

Step 1: Document Current Certificate Templates

Log in to the Source Server (Current CA host) as a member of the local administrator group.

Review Existing Certificate Templates:

Start > Administrative Tools > Certificate Authority on the Source Server (Current CA host).

In the Certification Authority window, expand the server node, and click on the Certificate Templates folder to view all the templates currently configured.

Document Template Settings:

Carefully note all certificate templates listed in the Certificate Templates folder.

Record any customized settings for each template, including template properties, validity periods, enrollment settings, and issuance requirements.

Important: Once the migration to the Destination Server (New CA host) is complete, you will need to manually configure the same set of certificate templates that were used on the Source Server (Current CA host) to maintain consistency in certificate issuance and management.

Step 2: Backup the Source Server (Current CA host) Certificate Authority Database and Configuration

Create a new folder CA2 in the root of C:\ drive - C:\CA2

Navigate to Certificate Authority:

Start > Administrative Tools > Certificate Authority.

Initiate Backup:

Right-click on the Server Node and select All Tasks > Backup CA to start the Certification Authority Backup Wizard

Ensure both checkboxes are selected to back up both the CA database and configuration.

Choose a backup destination path C:\CA2 to store the backup file.

Click Next and provide a password to protect the private key and CA certificate file.

Click Finish to complete the backup process.

Step 3: Backup CA Registry Settings

Access Registry Editor:

Start > Run, type regedit, and click OK.

Export Registry Settings:

Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\CertSvc.

Right-click on the Configuration key and select Export.

Save the file with a unique name and store it in C:\CA2.

Step 4: Transfer Certificate Authority Database, Configuration, and Registry Settings

Copy your Certificate Authority Database, Configuration, and Registry Settings Folder C:\CA2 from your Source Server to your Destination Server \\<insert server hostname>\C$

Step 5: Uninstall the CA Service from Source Server (Current CA host)

Open Server Manager and select Remove Roles under Roles Summary.

Launch the Remove Roles Wizard:

Uncheck the Active Directory Certificate Services checkbox.

Click Next, then Remove Features.

If prompted to stop Internet Information Services (IIS), click OK.

Complete the Uninstallation

Restart the server to finalize the uninstallation of the CA service.

Step 6: Install Certificate Services on the Destination Server (New CA host)

Log in to the Destination Server (New CA host) as a Domain Administrator or a member of the local administrator group.

Open Server Manager and select > Add Roles and Features

Launch the Add Roles and Features Wizard:

Select Active Directory Certificate Services and click Next.

Select any required features and click Next.

Select Role Services:

Select Certification Authority and click Next.

Click Install to begin the process.

Once installation is complete, click Close.

Step 7: Configure Active Directory Certificate Services

Navigate to AD CS Configuration Server Manager > AD CS.

Start the Configuration Wizard

Configure Active Directory Certificate Services

Ensure the Enterprise Administrator credentials are displayed, then click Next.

Select Certification Authority and click Next.

Choose CA Type:

Ensure Enterprise CA is selected, then click Next.

Select Root CA as the CA type, then click Next.

Use Existing Private Key:

Select Use existing private key and import the previously backed-up certificate and private key and click Next.

Import Existing Certificate

Click Import, browse to the backup location from source server C:\CA2, select the private key file, and enter the password used for protection during backup.

Complete the Configuration:

Select the imported certificate, click Next, and leave the default certificate database path.

Click Configure to complete the configuration.

Stop Certificate Services:

Open an Administrator Command Prompt and run the following command to stop Certificate Services: net stop certsvc.

Modify the Registry:

Open the previously exported registry file in Notepad.

Change the CAServerName value to match the name of the Destination Server (New CA host).

Save the changes.

Step 8: Restore CA Backup

Navigate to Certification Authority:

Server Manager > Tools > Certification Authority.

Start the Certification Authority Restore Wizard:

Right-click on the server node > All Tasks > Restore CA.

Confirm the stop of Active Directory Certificate Services.

Restore the Backup:

Click Next to start the Certification Authority Restore Wizard.

Select both checkboxes to restore the database and configuration, and provide the backup file path.

Enter the password used to protect the private key and click Next.

Click Finish to complete the restore process.

Click Yes to restart the Active Directory Certificate Services.

Step 9: Restore the Registry Information

Restore the Registry Key:

Navigate to the location of the modified registry backup file and double-click to run the restore.

Confirm the registry restore by clicking Yes and then OK.

Step 10: Reissue Certificate Templates

Reissue Certificate Templates:

Server Manager > Tools > Certification Authority.

Right-click on the Certificate Templates Folder > New > Certificate Template to Reissue.

From the list of available certificate templates, select the appropriate ones that you recorded from Step 1 of this document and click OK.
